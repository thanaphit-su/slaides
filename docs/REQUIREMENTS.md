# SLAIDES — Requirements

Each requirement is tagged `FR-NNN` (functional) or `NFR-NNN` (non-functional) and carries a one-line acceptance criterion. v0.1 scope only.

---

## Functional requirements

### Identity & access
- **FR-001** — An instructor can sign in or sign up with email + password through Supabase Auth. ✅ Access/refresh pair issued; local `app_user.approval_status` gates approved-only routes.
- **FR-002** — An audience member can join with a session code or share link. ✅ Code `SLD-XXXX-XX` resolves to one active session.
- **FR-003** — A guest provides email + display name to join. ✅ Both required unless `anonymous=true`.
- **FR-004** — A guest may join anonymously. ✅ Display name field disabled; email stored only as `sha256(email + session_salt)`.
- **FR-005** — Sign-in surfaces a clear instructor / audience mode switch on the same page. ✅ Two-tab segmented control.

### Deck management
- **FR-010** — Instructors see a library of their decks on first login. ✅ Workspace screen lists all decks owned by user.
- **FR-011** — Instructor can create a new empty deck. ✅ `+ New deck` produces a deck with one empty slide.
- **FR-012** — Instructor can rename a deck inline. ✅ Click title in editor top bar to edit.
- **FR-013** — Instructor can delete a deck. ✅ From library overflow menu, confirm modal.
- **FR-014** — Instructor can duplicate a deck. ✅ From library overflow menu, instant copy.
- **FR-015** — Instructor can export a deck as a `.slaides` zip. ✅ Contains `deck.json`, `slides/*.md`, `widgets/*.html`, `assets/*`.
- **FR-016** — Instructor can import a `.slaides` zip. ✅ Drag-and-drop on library, file picker on toolbar.

### Editor
- **FR-020** — Each stored slide has its own markdown document, normally with one `#` title. ✅ User typing is stored verbatim; H1 auto-splitting is not run on save.
- **FR-021** — The editor has a collapsible left sidebar with `Sections` and `Theme` tabs. ✅ State persists per user.
- **FR-022** — Active slide is rendered in a single-column blog-like canvas. ✅ 920px max-width, generous whitespace.
- **FR-023** — Right-clicking the canvas opens a context menu with copy/cut/paste, generate widget, insert widget, edit as markdown. ✅ Default browser context menu suppressed.
- **FR-024** — With a text selection active, right-click also offers **Interpret with AI**. ✅ Visible only when `selection.length > 0`.
- **FR-025** — Interpret opens a popover with the selected text, a prompt field (default "in plain English"), and a response area. ✅ LLM call streamed; Insert below adds response as a new paragraph.
- **FR-026** — Generate widget opens an inline AI chat that produces a draft widget. ✅ Draft can be inserted, hand-edited, or saved to library.
- **FR-027** — The right-edge tab opens the widget collection sidebar. ✅ Persistent collapsible right sidebar (400-450px expanded, 31px collapsed pill)
- **FR-028** — A bottom stepper navigates between slides with prev/next + pip rail + counter. ✅ Pip click jumps to slide.
- **FR-029** — Hovering the top of the canvas reveals an "Add slide above" ribbon. ✅ 50–80% opacity, click inserts.
- **FR-030** — Theme tab lists at least one selectable theme (Editorial Press). Others may render as preview-only. ✅
- **FR-031** — Each slide can have **at most one widget** (1 widget per slide). The UI gates inserts when this is reached and surfaces an Adjust path instead. ✅
- **FR-032** — The slide canvas is `contentEditable`; selections show a floating WYSIWYG toolbar (bold/italic/underline, H1/H2/¶/quote, inline code, link, Interpret-with-AI). ✅ Toolbar repositions via `selectionchange` + `range.getBoundingClientRect()`.
- **FR-033** — Each widget has an Adjust panel with preview, editable code (HTML/JS/CSS with explicit Save), bound props, and an AI chat panel that produces natural-language edits to the widget code. ✅

### Widget library (Widgets v2)
- **FR-040** — The widget API payload is `{id, deck_id, derived_from_id, name, kind, description, html, js, css, props_schema, example_props, behavior, ai_spec, tags, version, current_revision_id}`. ✅ Database ownership is deck-local; `example_props` and `ai_spec` are revision-owned and flattened into `WidgetOut`
- **FR-040.1** — Widget revisions are tracked in `widget_revision` table with versioned source (html, js, css, props_schema, behavior, ai_spec). ✅ AI Adjust creates new revision; rollback supported
- **FR-040.2** — Widget AI conversations are persisted in `widget_ai_thread` and `widget_ai_message` tables. ✅ Thread hydrates prior messages when sidebar reopened
- **FR-041** — Widgets render inside a sandboxed iframe with a postMessage bridge. ✅ Bridge defines `window.slaides` (emit, on, setState, contribute, props, participant, api.llm)
- **FR-042** — Widgets can be inserted from deck library, generated by prompt, imported as `.swidget`, or copied from another deck. ✅ Cross-deck copy creates independent widget with `derived_from_id` lineage
- **FR-043** — Widgets can be exported individually as `.swidget` (HTML with frontmatter including behavior, ai_spec, example_props). ✅
- **FR-044** — Widgets in v0.1 use self-contained HTML/CSS/JS. ✅ Current generation contract rejects remote scripts/styles; image loads may use `data:` or `https:`.
- **FR-045** — Widgets belong to exactly one deck (deck-local ownership). ✅ Cross-deck reuse requires explicit copy operation; copy creates independent widget with soft `derived_from_id` pointer for lineage
- **FR-046** — Widgets declare behavior as Quiet (default) or Loud with aggregator. ✅ Quiet: local-only; Loud: audience aggregation via `window.slaides.contribute()` and `window.slaides.on('state', cb)`
- **FR-047** — Slide placements capture `revision_id` at attach time. ✅ Placements render historical widget revision, not mutable current
- **FR-048** — Placement props are customizable per slide via Props tab. ✅ Form rendered from `props_schema`; PATCH validated against schema

### Live sessions
- **FR-050** — Instructor can start a session from the editor top bar or settings drawer. ✅ Creates a `session` row + shareable code/URL.
- **FR-051** — Audience joins by code or URL and is placed on the session's current slide. ✅
- **FR-052** — Presenter sees a LIVE badge, audience count, pending-questions count. ✅
- **FR-053** — Presenter can advance or rewind slides; the audience view follows in <500ms. ✅ p95 latency budget.
- **FR-054** — Presenter can open an ad-hoc poll, open-question, or random-audience pick from the FAB (floating action button). ✅ Inserted as `session_slide` row (not deck mutation); native interactions use the session interaction endpoints/WS events, while Loud iframe widgets use `widget.contribute`.
- **FR-055** — Audience can vote on polls and reply to questions. ✅ Realtime stream of results.
- **FR-056** — Audience can raise a question with optional anonymous flag. ✅ Appears in presenter's right rail within 1s.
- **FR-057** — Presenter can mark a question answered (logs the timestamp). ✅
- **FR-058** — Presenter can jump to the slide context of a question. ✅ One click; audience follows.
- **FR-059** — Both audience and instructor can right-click selected slide text to invoke Interpret. ✅
- **FR-060** — Presenter can copy a shareable link from the top bar; success state confirms. ✅ 1.5s toast.

### Interaction logging & analytics (the transcript)
- **FR-070** — Every interaction is persisted to the `interaction_log` table with a `(session_id, participant_ref, slide_id, widget_id, kind, payload, occurred_at)` row. ✅
- **FR-071** — Anonymous interactions store `participant_ref = sha256(email + session_salt)` and **never** the raw email on the row. ✅
- **FR-072** — Registered audience interactions link by `participant_ref = user_id`. ✅
- **FR-073** — LLM calls, including Interpret, are logged to `llm_call` with `{purpose, model, prompt_hash, latency_ms, tokens_in, tokens_out}` and no raw prompt text. ✅
- **FR-074** — After a session ends, the instructor can view a transcript: chronological list + per-slide summary + per-participant summary. ✅ Shipped 2026-05-28 via `GET /api/v1/sessions/{id}/transcript`; UI at `apps/web/src/pages/SessionTranscript.vue` with a Presenter-faithful slide pane (using the persisted `placement_states[]` for Loud widgets and `session_slide.results` for native live interactions) plus a Timeline / Per-Slide / Participants sidebar.
- **FR-075** — The transcript shows interaction frequency, all Q&A. ✅ Shipped 2026-05-28 — per-slide aggregates (`interaction_count`, `by_kind`) and per-participant aggregates (`total_interactions`, `by_kind`) come from server-side grouped `func.count()` queries; questions surface as `question.raised` events in the timeline. Most-frequent terms in `llm.interpret` calls remain client-side analysis (selection / prompt are decrypted server-side before returning when the workspace opted into prompt logging via the new `workspace.log_llm_prompts_for_transcript` toggle).
- **FR-076** — Transcripts can be exported as CSV or `.slaides-session` JSON. ✅ Shipped 2026-05-28 via `GET /api/v1/sessions/{id}/transcript.csv` and `GET /api/v1/sessions/{id}/transcript.json` (10k-event cap with explicit `truncated` metadata in the export header).

### Settings
- **FR-080** — Workspace owner configures OpenAI-compatible endpoint with multi-model library. ✅ Per-workspace settings: base URL, encrypted API key, model library (with advanced params: max_context_window, max_output_tokens, temperature, top_p, frequency_penalty, presence_penalty, supports_image_input), capability-to-model routing
- **FR-081** — Workspace owner routes each AI capability to a model or disables it. ✅ Capabilities: `inline_write`, `interpret`, `widget_generate`; `None` disables capability
- **FR-082** — API keys are encrypted at rest using Fernet encryption. ✅ LLM_ENCRYPTION_SECRET or JWT_SECRET fallback for local dev
- **FR-084** — Dark mode toggle persists per-user. [BACKLOG] Later release; current Settings UI labels this as not implemented.
- **FR-085** — Editor density (Comfortable / Compact / Reading) persists per-user. [BACKLOG] Later release; current Settings UI labels this as not implemented.

### Widget AI workflow
- **FR-090** — Widget Generate uses structured AI workflow with clarification questions. ✅ Behavior choice (Quiet/Loud) through option chips, not manual picker
- **FR-091** — Widget Generate streams with live feedback. ✅ Typing dots, character counter, stream tail preview
- **FR-092** — Widget Adjust can swap behavior in either direction (Quiet ↔ Loud). ✅ PATCH sends only changed draft fields, including `behavior`/`kind` when intentionally changed, and may require reset confirmation for active placement state.
- **FR-093** — AI workflow envelopes include: `question`, `draft`, `plan`, `step`, `reflection`. ✅ Parser fails closed on legacy flat JSON
- **FR-094** — Post-stream validators run on generated widgets. ✅ `_scan_theme_violations`, `_scan_layout_violations`, `_scan_props_contract_violations`, `_scan_behavior_violations`
- **FR-095** — Widget AI threads persist per widget. ✅ `widget_ai_thread` / `widget_ai_message` tables; adjust mode hydrates history

## Non-functional requirements

### Performance
- **NFR-001** — First contentful paint on the editor: ≤ 1.5s p95 on a warm connection.
- **NFR-002** — Slide transition (instructor → audience displayed): ≤ 500ms p95.
- **NFR-003** — Live poll round-trip (vote → broadcast → render): ≤ 700ms p95.
- **NFR-004** — Markdown render of a single slide: ≤ 50ms.
- **NFR-005** — Widget iframe boot: ≤ 250ms (including bridge install).

### Scale (v0.1 targets)
- **NFR-010** — A single session supports up to **500 concurrent audience members**.
- **NFR-011** — A workspace supports up to **500 decks** and **2,000 widgets**.
- **NFR-012** — Interaction log retention: 90 days minimum.

### Reliability
- **NFR-020** — Audience reconnect after a 30s network drop without losing the current slide / interaction state.
- **NFR-021** — A widget crash never crashes the deck (sandbox isolation).
- **NFR-022** — A session continues to deliver audience interactions if the LLM proxy is down (LLM features degrade gracefully).

### Security & privacy
- **NFR-030** — All traffic over TLS 1.3.
- **NFR-031** — Per-workspace LLM API keys are encrypted at rest with a workspace-scoped envelope key. ✅ Fernet encryption via LLM_ENCRYPTION_SECRET
- **NFR-032** — Widget iframes use `sandbox="allow-scripts allow-forms"` with `srcdoc` and a sandbox-null origin. ✅ The srcdoc CSP blocks network fetches except allowed image loads (`img-src data: https:`).
- **NFR-033** — Guest JWTs are session-scoped with `sid` + `ref` claims. ✅ Rejects on session_id mismatch; expires with session
- **NFR-034** — Anonymous participant emails are hashed into `participant_ref = sha256(email + per-session-salt)`; the salt is rotated each session.
- **NFR-035** — Cleartext PII is never stored on an `interaction_log` row.
- **NFR-036** — Widget iframes do not use `allow-same-origin` or `allow-top-navigation`.
- **NFR-037** — Widget srcdoc CSP allows inline widget scripts, disallows `connect-src`, and permits `img-src data: https:` for image-driven widgets.
- **NFR-038** — Rate limit: 60 LLM calls/minute per workspace; 6 widget-generate calls/minute per user.

### Accessibility
- **NFR-040** — All interactive elements reachable by keyboard.
- **NFR-041** — Focus rings visible (we use a 3px accent-soft halo).
- **NFR-042** — Color contrast ≥ AA on all text. Verified for both light and dark themes.
- **NFR-043** — All icon-only buttons carry a tooltip + `aria-label`.
- **NFR-044** — Live regions on the presenter's question rail and audience interaction inserts.

### Browser support
- **NFR-050** — Chrome / Edge / Safari / Firefox — last two stable versions.
- **NFR-051** — Mobile Safari + Chrome on Android — last two stable versions, audience view only.

### Internationalisation (deferred but not blocked)
- **NFR-060** — All UI copy lives in a single message catalog file; English in v0.1. [BACKLOG] Deferred; current UI copy is colocated in Vue components.

## Out of scope (explicit)

Listed so the implementing team doesn't assume:

- Native mobile apps (iOS / Android).
- Team collaboration on a single deck (multi-cursor, presence).
- Slide transitions / animations beyond fade-in.
- Image / video / audio upload in markdown.
- Paid plans, Stripe, invoicing.
- Public widget marketplace.
- SCIM / SSO / enterprise admin.
- Per-deck LLM keys (only per-workspace in v0.1).
- Localization of the SLAIDES chrome (English-only).
